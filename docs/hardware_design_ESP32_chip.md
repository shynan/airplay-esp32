# ESP32 裸芯片硬件设计方案

## 一、概述

本方案使用 ESP32 裸芯片（QFN-48封装），配合外置 Flash 和 PSRAM，适用于：
- 需要控制成本的大批量生产
- 需要定制 PCB 尺寸
- 需要优化射频性能

---

## 二、核心器件选型

| 器件 | 型号 | 封装 | 规格 | 数量 |
|------|------|------|------|------|
| 主控芯片 | ESP32-D0WDQ6 | QFN-48 | 双核 240MHz, WiFi+BT | 1 |
| Flash | W25Q64JVSIQ | SOIC-8 | 8MB, 133MHz, QSPI | 1 |
| PSRAM | ESP-PSRAM32 | SOP-8 | 4MB, 8MHz, Pseudo-SRAM | 1 |
| 时钟芯片 | Si5351A | MSOP-10 | 可编程时钟发生器 | 1 |
| DAC | ES9018K2M | QFN-28 | 32bit/384kHz DAC | 1 |
| 运放 | OPA1612A | SOIC-8 | 低噪声音频运放 | 1 |

### 4.2 GPIO 冲突与调整

由于 Flash 和 PSRAM 占用引脚，需调整部分设计：

```
原设计 → 调整后设计:

I2C0_SDA: GPIO2 → GPIO4  (GPIO2是Strapping引脚)
I2C0_SCL: GPIO4 → GPIO5  (避免冲突)
DAC_RESET: GPIO5 → GPIO33
LED_RGB: GPIO32 → GPIO25 (或与屏幕复用)
```

## 七、Strapping 引脚配置

### 7.1 Strapping 引脚详解

```
                    Strapping 引脚配置
                    ───────────────────

    ESP32 上电时会采样以下引脚决定启动模式:

    ┌─────────┬─────────────┬───────────────────────────────────────────┐
    │  引脚   │  默认状态    │  功能说明                                  │
    ├─────────┼─────────────┼───────────────────────────────────────────┤
    │ GPIO0   │ 上拉 (HIGH)  │ LOW = 下载模式 (进入Bootloader)           │
    │         │             │ HIGH = 正常运行 (从Flash启动)              │
    ├─────────┼─────────────┼───────────────────────────────────────────┤
    │ GPIO2   │ 悬空/上拉    │ LOW = 禁用调试日志输出                     │
    │         │             │ HIGH = 启用调试日志                        │
    ├─────────┼─────────────┼───────────────────────────────────────────┤
    │ GPIO12  │ 下拉 (LOW)   │ LOW = Flash 3.3V (推荐)                   │
    │         │             │ HIGH = Flash 1.8V (需特殊Flash芯片)        │
    ├─────────┼─────────────┼───────────────────────────────────────────┤
    │ GPIO15  │ 上拉 (HIGH)  │ LOW = 启动时输出调试日志                   │
    │         │             │ HIGH = 禁用启动日志 (正常启动)             │
    └─────────┴─────────────┴───────────────────────────────────────────┘


    正确的 Strapping 电路:
    ──────────────────────────────────────────────────────────────────────

    GPIO0 (BOOT按键):
    ──────────────────────────────────────────────────────────────────────

         3.3V
           │
          10K (上拉电阻)
           │
    GPIO0 ─┼────────┬────────► 正常运行模式
           │        │
           │       ═══ 100pF (消抖)
           │        │
          按键     GND
           │
           └────────┴────────► 按下时进入下载模式

    注意: 按键按下时 GPIO0 = LOW，进入下载模式


    GPIO12 (Flash电压选择) - 必须下拉:
    ──────────────────────────────────────────────────────────────────────

         3.3V
           │
          NC (不连接)
           │
    GPIO12 ─┼────────────────► 必须为LOW
           │
          10K (下拉电阻)
           │
          GND

    ⚠️ 重要: GPIO12 必须下拉！否则可能无法正常启动！


    GPIO15 (启动日志控制):
    ──────────────────────────────────────────────────────────────────────

         3.3V
           │
          10K (上拉电阻)
           │
    GPIO15 ─┼────────────────► HIGH = 正常启动，无调试日志
           │
           │
           │
          GND (可选，用于调试)


    GPIO2 (调试日志控制):
    ──────────────────────────────────────────────────────────────────────

         3.3V
           │
          10K (上拉电阻，可选)
           │
    GPIO2  ─┼────────────────► 可浮空或上拉

    注意: 如果使用 I2C，需确保此引脚状态不影响启动
```

```
    | 模块方案 | ¥55   | 设计简单，RF性能有保证    | 成本略高              │
    | 芯片方案 | ¥61   | 理论成本更低，灵活度高    | 设计复杂，需调试RF    |

    实际上，芯片方案由于需要更复杂的PCB和调试，总成本可能更高。
    建议: 小批量用模块，大批量(>10000)用芯片方案。
```

### 12.2 常见问题

```
    问题1: 无法启动，不断重启
    ─────────────────────────────────────────────────────────────────────
    原因: GPIO12 未下拉，Flash电压选择错误
    解决: 确保 GPIO12 通过 10K 下拉到 GND

    问题2: Flash 检测失败
    ─────────────────────────────────────────────────────────────────────
    原因: Flash 焊接问题或走线过长
    解决:
    1. 检查 Flash 焊接
    2. 检查 SPI 走线阻抗
    3. 添加串联电阻 (22Ω) 匹配阻抗

    问题3: PSRAM 初始化失败
    ─────────────────────────────────────────────────────────────────────
    原因: PSRAM 时序问题
    解决:
    1. 降低 PSRAM 时钟频率
    2. 检查 PSRAM 焊接
    3. 确认 PSRAM 型号 (ESP-PSRAM32 vs ESP-PSRAM64)

    问题4: WiFi 性能差
    ─────────────────────────────────────────────────────────────────────
    原因: 天线匹配不佳或净空区不够
    解决:
    1. 使用网分调试天线匹配
    2. 确保天线净空区足够
    3. 检查地平面完整性
```
